# IMRAD Structure Reference

## Introduction

### Purpose
Establish context, motivate the research, and present the paper's direction.

### Subsections
1. **Background** - General area context (2-3 paragraphs)
2. **Problem Statement** - Specific problem being addressed
3. **Literature Review** - Prior work, current state, identified gaps
4. **Research Questions/Objectives** - What this paper investigates
5. **Contributions** - Novel aspects of this work
6. **Paper Structure** - Overview of remaining sections

### Length
Typically 500-1000 words (journal-dependent)

---

## Methodology

### Purpose
Describe the research design, data collection, and analysis methods.

### Subsections (Empirical Research)
1. **Research Design** - Overall approach (quantitative/qualitative/mixed)
2. **Data Collection** - Subjects, instruments, procedures
3. **Data Analysis** - Statistical or analytical methods
4. **Validity/Rigor** - Measures to ensure accuracy

### Subsections (Technical/Engineering)
1. **System Overview** - High-level architecture
2. **Method/Algorithm** - Core technical approach
3. **Implementation Details** - Key design choices
4. **Evaluation Setup** - How system was tested

### Length
Typically 800-1500 words

---

## Results

### Purpose
Present findings objectively without interpretation.

### Subsections
1. **Descriptive Statistics** - Overview of data/results
2. **Main Findings** - Primary results with supporting data
3. **Secondary Findings** - Additional interesting results
4. **Statistical Analysis** - Significance testing results

### Key Principles
- Present data in logical sequence
- Use tables/figures to summarize key results
- Report, don't interpret
- Include effect sizes and confidence intervals

### Length
Typically 1000-2000 words

---

## Discussion

### Purpose
Interpret results, explain significance, acknowledge limitations.

### Subsections
1. **Interpretation** - What results mean
2. **Comparison** - How results align with/differ from prior work
3. **Implications** - Practical/theoretical significance
4. **Limitations** - Constraints on validity
5. **Future Work** - Open questions and next steps

### Length
Typically 800-1500 words

---

## Conclusion

### Purpose
Summarize contributions and their significance.

### Subsections
1. **Summary** - Brief recap of research and findings
2. **Contributions** - Key novel contributions
3. **Implications** - Practical impact
4. **Closing** - Final thoughts

### Key Principles
- Be concise (typically 200-500 words)
- Don't introduce new information
- Emphasize significance

---

## Variation: Engineering Paper Structure

For engineering/technical papers, IMRAD adapts as follows:

```
1. Introduction (includes brief Related Work or separate as 1.5)
2. Proposed Method / System Design
3. Experiments / Evaluation
4. Discussion
5. Conclusion
```

**Note**: "Related Work" can be a subsection of Introduction (1.1-1.2) for short papers, or a standalone section (Section 2) for comprehensive literature reviews within engineering papers.

## Variation: Literature Review / Systematic Review

```
1. Introduction
2. Search Methodology (include PRISMA flow diagram)
3. Thematic Analysis / Synthesis
4. Findings
5. Discussion
6. Conclusion
```

**PRISMA Checklist**: For systematic reviews, follow PRISMA guidelines and include the flow diagram showing study selection process.

## Additional Required Sections (Most Elsevier Journals)

Depending on journal requirements, may need:

### Conflict of Interest Statement
- Required for all Elsevier journals
- Standard text: "The authors declare that they have no known competing financial interests..."

### Funding Statement
- Required if any funding was received
- Format: "This work was supported by [funder] under grant [number]"

### Author Contributions
- Required by Nature, PLOS, and increasingly common
- Use CRediT taxonomy: Conceptualization, Methodology, Software, Validation, etc.

### Data Availability Statement
- Required by many journals for reproducibility
- Specify where data is available (repository, supplemental files, etc.)

### Ethical Approval / Ethics Statement
- Required for research involving humans, animals, or sensitive data
- Include IRB/ethics committee approval number

### Highlights (Elsevier specific)
- 3-5 bullet points summarizing paper's contributions
- Maximum 85 characters per point
- Placed before abstract in final manuscript

### Supplementary Materials
- Referenced as "Appendix A", "Supplementary Figure S1", etc.
- Not essential for main argument

## Section Length Guidelines

| Section | Words (typical) |
|---------|-----------------|
| Abstract | 150-300 |
| Introduction | 500-1000 |
| Methodology | 800-1500 |
| Results | 1000-2000 |
| Discussion | 800-1500 |
| Conclusion | 200-500 |
